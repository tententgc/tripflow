# .impeccable.md — TripFlow Design Context

## Design Context

### Users
- **Primary**: Thai travelers (age 25-55) on group tours worldwide, using the PWA during trips — often in unfamiliar countries, sometimes without reliable internet (especially in China)
- **Secondary**: Thai tour operators managing trips via the admin portal
- **Context**: Users open the app many times a day while traveling — checking today's schedule, finding the hotel address, looking up emergency contacts, splitting dinner bills. They're often tired, jet-lagged, or in a hurry. The app must be instantly legible and calming, not demanding.
- **Job to be done**: "I want to know exactly what's happening today and feel confident I won't miss anything on this trip."

### Brand Personality
**Modern, Confident, Smart**

- Voice: Clear, direct, helpful — like a sharp travel companion, not a corporate concierge
- Tone: Cool confidence with moments of warmth. Professional but never cold
- Emotional goals: In control, informed, delighted by good design — travelers should feel the app is smarter than other travel tools

### Aesthetic Direction

**Visual Tone**: iOS 18 meets fintech — bento grid layout with varied card sizes, information-dense but visually clean. Data-forward design where every widget earns its space.

**Theme**: Light mode. Cool white base (`#f8fafc`) with indigo (`#4f46e5`) as primary and coral (`#f97066`) as energetic secondary. Clean, modern, tech-travel fusion.

**References**:
- **iOS 18 Widgets** — varied card sizes in a bento grid, rounded corners, at-a-glance info density, subtle shadows
- **Monzo / N26 banking** — clean transaction cards, spending insights, beautiful data visualization, card-based navigation
- **Stripe Dashboard** — information hierarchy, clean typography, purposeful use of color for data

**Anti-references** (must NOT look like):
- Cluttered travel agency websites — no busy banners, flashing deals, info overload
- Corporate enterprise dashboards — no stiff tables, no personality-free UI
- Overly playful / cartoon — no excessive illustrations, childish animations
- Flat / generic — no Bootstrap-default feel, no unstyled components

### Design Principles

1. **Bento Intelligence** — Every card is a widget with a purpose. Varied sizes communicate hierarchy: large cards for active trips, medium for today's schedule, small for quick-glance data (weather, currency, countdown). The grid itself tells the story.

2. **Data as Delight** — Numbers, progress bars, charts, and status indicators are first-class UI elements. Show trip progress (Day 3/7, 42%), spending breakdowns, countdown timers. Make data beautiful, not just readable.

3. **Glanceable First** — Travelers check the app in 3-second bursts. Key info (time, place, meal) must be scannable without reading. Use icons, color coding, and spatial hierarchy within bento cards.

4. **Touch-Friendly Always** — 44px minimum targets, generous spacing, forgiving hit areas. Users are walking, on buses, holding bags. Fat-finger tolerance is a feature.

5. **Cool Confidence** — The interface radiates competence through restraint. Indigo conveys trust and intelligence. Coral adds energy at key moments (CTAs, alerts, highlights). White space is generous but purposeful.

### Component Patterns

- **Bento Grid**: CSS Grid with `auto-rows` and `span` variants. Cards span 1x1 (small widget), 2x1 (wide), 1x2 (tall), or 2x2 (hero). Gap 12-16px. Responsive: 2-col on mobile, 3-4 col on tablet+.
- **Widget Cards**: White surface, 20-24px radius, subtle border (`slate-200`) + shadow-md. Optional tinted backgrounds (`indigo-50`, coral tint) for emphasis.
- **Navigation**: Glassmorphic bottom nav, indigo active states (not orange). Dot indicator under active tab.
- **Badges**: Pill-shaped with tinted backgrounds. Category colors: sightseeing=indigo, food=coral, transport=slate, accommodation=purple, shopping=pink, temple=amber, nature=emerald.
- **Buttons**: Primary = indigo fill + indigo glow shadow. Secondary = coral fill + coral glow. Ghost = transparent + border. Hover: slight lift (`translateY(-1px)`). Press: `scale(0.98)`.
- **Progress Indicators**: Rounded progress bars with indigo fill. Circular progress for trip completion. Animated number counters for stats.
- **Data Visualization**: Mini bar charts, donut charts for spending, sparklines for trends — all using the indigo/coral palette.
- **Overlays**: Bottom sheets on mobile (drag handle, 24px radius top), centered modals on desktop.
- **Animations**: Staggered fade-in for bento cards on load. Subtle hover lift on cards. `prefers-reduced-motion` respected everywhere. No spring physics — CSS transitions only (200-300ms ease-out).
- **Activity Category Colors**: Consistent across all views:
  - Sightseeing: `#4f46e5` (indigo)
  - Food: `#f97066` (coral)
  - Transport: `#64748b` (slate)
  - Accommodation: `#a855f7` (purple)
  - Shopping: `#ec4899` (pink)
  - Temple: `#f59e0b` (amber)
  - Nature: `#10b981` (emerald)
  - Nightlife: `#8b5cf6` (violet)

### Migration Notes (from v1 warm theme)

When updating existing components from the previous warm design:
- `#faf8f5` (warm cream) → `#f8fafc` (cool slate-50)
- `#2c1810` (warm brown text) → `#1e293b` (slate-800)
- `#7c5a3e` / `#b8a394` (warm muted) → `#475569` / `#94a3b8` (slate-600/400)
- `#f97316` (orange accent) → `#4f46e5` (indigo primary) or `#f97066` (coral secondary)
- `rgba(234,88,12,0.12)` (orange shadow) → `rgba(79,70,229,0.15)` (indigo shadow)
- `rgba(44,24,16,...)` (warm overlay) → `rgba(15,23,42,...)` (slate-900 overlay)
- Orange gradients → Indigo-to-violet gradients (`from-indigo-600 to-violet-600`)
- Card radius 18-22px → 20-24px (bento cards) or 16px (inner cards)
